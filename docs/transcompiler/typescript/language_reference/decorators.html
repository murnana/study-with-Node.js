デコレ―タ

デコレ―タ

またはAngulerなどで見かける「@Component」について。

概要

(TypeScript2.9現在) 実験的な機能らしい。Angulerでなぜ使われたし

関数の中で関数を生成するとか、クラスに新たな宣言をつけるといった使い方ができる。 C#を知っているなら、拡張メソッドや属性あたりを思えばいいかもしれない。

使用したい場合は、TypeScriptコンパイル時にオプションを渡すか、tsconfigで設定する必要がある。

Command Line:

tsc --target ES5 --experimentalDecorators

tsconfig.json:

{
    "compilerOptions": {
        "target": "ES5",
        "experimentalDecorators": true
    }
}

デコレ―タの作り方

デコレ―タが宣言されたものに対して、どのように働くのかを記述する方法は、関数宣言と同じ。 記述されたものは「デコレ―タファクトリー」。 デコレ―タファクトリーによって生成されたもの(たいていは関数)を「デコレ―タ」という。

1行の場合:

@f @g x

複数行の場合:

@f
@g
x

クラスデコレ―タ

コンストラクタに適用される。 以下はGreeterクラスのコンストラクタとプロトタイプを密封する例。 C#のsealedみたいなことができる？

function sealed(constructor: Function) {
    Object.seal(constructor);
    Object.seal(constructor.prototype);
}

@sealed
class Greeter {
    greeting: string;
    constructor(message: string) {
        this.greeting = message;
    }
    greet() {
        return "Hello, " + this.greeting;
    }
}
